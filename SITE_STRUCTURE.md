## ページ一覧

### 1. TOP（/）- 判断LP

**役割**: 判断会社としてのポジショニングを伝える入口

#### セクション構成

1. **ヒーローセクション**
   - 見出し: 「何から手をつければいいかわからない」
   - 副見出し: そんな中小企業のための会社です。
   - 説明: WebもSNSもDXも、選択肢が多すぎて決められない。その状態から一緒に整理します。
   - CTA: 「何から手をつけるべきか相談する」

2. **問題の再定義**
   - 見出し: 「何が正解か」を判断できていないことです。
   - 説明: 問題は「施策が分からない」ことではありません。

3. **DP-GUILDの立ち位置**
   - 見出し: 最初の判断を一緒に決める会社です。
   - 説明: 作る会社ではありません。やるべきか、やらないべきか。その判断から一緒に考えます。
   - 補足: 「とりあえずホームページを作る」ではなく、「本当に今ホームページが必要か」から始めます。

4. **支援の流れ（Process）**
   - 01 相談: 現状と制約を整理
   - 02 分析: 優先度と実現性を見極める
   - 03 判断: やること／やらないことを決める
   - 04 実行: 決めたことを形にする
   - 05 改善: 結果を見て次を判断する

5. **判断の結果（What Happens）**
   - WebサイトやLPを作り直す判断をする
   - SNSは今やらない、という判断をする
   - 社内で回せる仕組みを整える
   - 外注せず、自走する選択をする

6. **向いている方 / 向いていない方（Fit Check）**

   **向いている方:**
   - WebやSNSに課題は感じているが、何から始めるべきか分からない
   - 施策の前に、考え方や優先順位を整理したい
   - 単発ではなく、継続的に相談できる関係がほしい

   **向いていない方:**
   - とにかく安く・早く作りたい
   - 判断は自分でするから、言われた通りに作業してほしい

7. **実績（Track Record）**
   - 継続契約の解約: 0件
   - 説明: 判断と改善を繰り返す関係だから、続いています。

8. **CTA**
   - 見出し: 「何から相談すればいいか分からない」それで大丈夫です。
   - CTA: 「まず状況を整理する相談をする」

9. **裏取り導線**
   - 会社概要 / 実績・事例 / ブログ

10. **フッター（3カラム形式）**

### 3. 判断の記録（/results）

**役割**: 判断プロセスの具体例を公開し、判断会社としての実績を証明

#### セクション構成

1. **ヒーロー**
   - 見出し: 判断の記録
   - 副見出し: 「何を選び、何を選ばなかったか」お客様と一緒に考えた判断プロセスを公開しています。

2. **判断ログ（6事例）**

   各事例のフォーマット:
   - 相談内容
   - 検討した選択肢（A/B/C）
   - やらなかった判断（理由）
   - 最終判断
   - その後

   **判断事例01**: 製造業（醸造業者）
   - 相談: 在庫管理が非効率で、顧客満足度も上げたいが、予算が限られている
   - 選択肢: A:高機能在庫管理システム / B:シンプル在庫管理+Wi-Fi / C:SNS全振り
   - やらなかった: A（機能8割使わない、運用負荷リスク）
   - 最終判断: Bを基盤に、余った予算でCを小規模実施
   - その後: 在庫ロス減少、リピーター増加

   **判断事例02**: サービス業（ストレッチスタジオ）
   - 相談: 新規開業。予算が限られている中で集客と信頼性を両立したい
   - 選択肢: A:フル機能サイト+予約システム / B:最低限サイト+外部予約連携 / C:SNSのみ
   - やらなかった: A（過剰投資リスク）
   - 最終判断: Bを選択
   - その後: 集客力向上、予算内で安定経営基盤

   **判断事例03**: 製造業（忍者服工房）
   - 相談: 伝統的な忍者服をECで販売したいが、どこまで作り込むべきか
   - 選択肢: A:Shopify等 / B:フルスクラッチ / C:BASE等で様子見
   - やらなかった: A（テンプレートでは世界観を表現しきれない）
   - 最終判断: Bを選択（シンプル設計でフルスクラッチ）
   - その後: 訪問者数・売上増加、ファン定着

   **判断事例04**: 医療（リハビリ業者）
   - 相談: 脳疾患特化型リハビリの認知が低い。Web集客を強化したい
   - 選択肢: A:広告集中投下 / B:LP+SEO / C:SNS認知拡大
   - やらなかった: A（受け皿がないと広告が無駄）
   - 最終判断: Bを選択
   - その後: Web問い合わせ大幅増加、CV率向上

   **判断事例05**: 教育（デザインスクール）
   - 相談: Instagram活用で認知を広げたいが、効果的なコンテンツがわからない
   - 選択肢: A:作品紹介中心 / B:教育動画でバイラル / C:広告中心
   - やらなかった: A（フォロワー外リーチが限定的）
   - 最終判断: Bを選択
   - その後: 動画1,600万回再生達成、受講生獲得大幅増加

   **判断事例06**: 製造業（大手食品メーカー）
   - 相談: データドリブンマーケティングを始めたいが、ベンダー選定がわからない
   - 選択肢: A:大手ベンダー一択 / B:要件明確化後に複数比較 / C:内製化
   - やらなかった: A（要件曖昧なまま発注すると後から問題発生）
   - 最終判断: Bを選択（要求定義→RFP作成→複数社比較）
   - その後: 最適なベンダー選定成功
